# #148 fiche personnage — PR 1 : enrichissement (tags, background, secrets, colonnes)

## Objectif

Trois volets à faible risque de l'issue #148, regroupés :
- **① Tags affichés** : le champ `tags` (M2M) est éditable mais n'apparaissait pas sur la fiche.
- **④ Background + secrets** : en plus de `description`, deux champs texte — `background` (public)
  et `secrets` (réservé créateur/propriétaire, jamais public ni fédéré).
- **⑥ Colonnes desktop** : la fiche était mono-colonne ; passage en 2 colonnes (`@lg:grid-cols-3`)
  pour la lisibilité, en respectant la règle mobile-first (`@container app`, jamais `@media`).

Volets **hors PR 1** (structurels) : ③ édition des traits/actions existants, ② bouton « valider »,
⑤ conséquences conditionnelles → PR 2/3.

## Contexte technique vérifié

| Site | Action |
|------|--------|
| `Character` (`models.py`) | Ajout `background` + `secrets` (`TextField(blank=True)`) après `description` |
| Migration | `0022_character_background_character_secrets` (2 `AddField`, non destructive) |
| Service `create_character_with_sheet` | Params `background`/`secrets` (défaut `""`), posés sur `Character(...)` |
| Vue `character_create` | Lit `background`/`secrets` du POST, passe au service |
| Vue `character_edit` | Lit + sauve (`update_fields` étendu) |
| `character_create.html` · `character_form.html` | Textareas background + secrets après description |
| `detail.html` | Puces tags (après en-tête) ; carte Background (public) ; carte Secrets (gardée `can_edit_traits`) ; grille 2 colonnes (narratif à gauche, blocs secondaires à droite) |

`can_edit_traits` (créateur ou propriétaire, déjà dans le contexte) sert de garde d'affichage des
secrets — pas de nouveau prédicat.

## Sécurité / fédération

- **`secrets` ne fuit jamais** : les serializers `CharacterSerializer`/`CharacterSearchSerializer`
  (DRF) listent des champs explicites — `background`/`secrets` absents. Le serializer AP n'y touche
  pas non plus. Vérifié par grep : aucun `fields = "__all__"`.
- Affichage `secrets` gardé côté template ET la donnée n'est pas exposée en API.

## i18n

Nouveaux msgids : « Background », « Secrets », « Public backstory shown on the sheet. »,
« Visible only to you (creator/owner). Never shown publicly. », « Only you (creator/owner) can see
this. ». fr sur une seule ligne (contrainte `test_no_empty_msgstr_in_fr_po`), `.mo` recompilés.

- Vue edit : persiste background/secrets.
- Fiche : tags + background visibles publiquement ; secrets visible au mainteneur, masqué aux
  autres et aux anonymes.
